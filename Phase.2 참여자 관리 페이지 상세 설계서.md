# 참여자 관리 페이지 상세 설계서

## Phase. 2 — `/participants`

---

### 1. 페이지 전체 구조

- **URL:** `/participants`
- **레이아웃:** 좌측 사이드바(네비게이션) + 우측 메인 콘텐츠
- **메인 콘텐츠 구성:**
  - 상단 헤더 영역
  - 필터/검색 영역
  - 참여자 테이블 영역
  - (오버레이) 사이드 드로어 — 참여자 선택 시 우측에서 슬라이드
- **진입 경로:**
  - ① 사이드바 네비게이션 직접 클릭
  - ② 기업 드로어 내 참여자 Popover의 `[상세보기 →]` 링크 (해당 참여자 드로어 자동 오픈)

### 2. 상단 헤더 영역

- **페이지 제목:** "참여자 관리"
  - 좌측 정렬
  - Typography: h2
- **우측 액션 버튼:**
  - `[+ 참여자 추가]` 버튼
    - Primary 스타일 (녹색 배경 + 흰 텍스트)
    - 클릭 시 → 참여자 추가 모달 오픈 (섹션 7 참조)

### 3. 필터 및 검색 영역

- **필터 탭 (좌측):**
  - `[전체]` `[훈련비]` `[지원비]` `[세미나]`
  - 동작: 탭 클릭 시 해당 과정 구분에 참여 중인 참여자만 필터링
  - 활성 탭: 하단 보더 또는 배경색 강조
  - 기본 선택: `[전체]`

- **보조 필터 (탭 우측, 드롭다운):**
  - `[수료 상태 ▼]` — 전체 / 수료 / 미수료
  - `[고용보험 ▼]` — 전체 / 가입 / 미가입
  - 필터 활성 시: 드롭다운 옆에 `✕` 초기화 버튼 표시

- **검색창 (우측):**
  - Placeholder: "이름, 기업명, 이메일, 연락처 검색"
  - 실시간 필터링 (debounce 300ms)
  - 좌측에 돋보기 아이콘

### 4. 참여자 테이블

#### 4-1. 컬럼 정의

| 순서 | 컬럼       | 너비      | 설명                                                      |
| ---- | ---------- | --------- | --------------------------------------------------------- |
| 0    | ☐ 체크박스 | 40px 고정 | 행 선택. 헤더 체크박스 = 전체 선택/해제                   |
| 1    | 이름       | 120px     | 클릭 시 사이드 드로어 열림                                |
| 2    | 소속 기업  | flex 1    | 기업명 표시, 클릭 시 `/companies` 해당 기업 드로어로 이동 |
| 3    | 직위       | 100px     | 직위/직급                                                 |
| 4    | 연락처     | 140px     | 전화번호                                                  |
| 5    | 이메일     | flex 1    | 이메일 주소 (truncate)                                    |
| 6    | 수료 현황  | 120px     | 진행률 표시 (예: `2/3 과정`)                              |

#### 4-2. 셀 표시 규칙

- **이름:**
  - 폰트: 14px, semibold
  - 색상: `text-gray-900`
  - hover 시: 밑줄 (링크 느낌)
- **소속 기업:**
  - 폰트: 14px, regular
  - 색상: `text-blue-600` (링크 스타일)
  - 클릭 시: `/companies` 페이지로 이동 + 해당 기업 드로어 자동 오픈
  - 기업 미등록 참여자: `—` 표시 (회색)
- **직위:**
  - 폰트: 14px, regular, `text-gray-600`
  - 빈 값일 경우: `—`
- **수료 현황:**
  - `수료 완료 수 / 전체 등록 과정 수` 형식
  - 전체 수료 시: `3/3` + 녹색 텍스트
  - 부분 수료: `2/3` + 기본 텍스트
  - 미수료: `0/3` + 회색 텍스트
  - 과정 미등록: `—`
  - **미니 프로그레스 바** (선택):

    ```
    2/3 과정  ██████░░░
    ```

    - 높이 4px, 텍스트 좌측 또는 하단에 배치
    - `green-500` (완료) + `gray-200` (잔여)

#### 4-3. 행 인터랙션

- **Hover:**
  - 행 전체 배경색 변경 (light gray)
  - cursor: pointer
- **Click:**
  - 행 클릭 → 해당 참여자의 사이드 드로어 오픈
  - 클릭된 행에 좌측 보더 강조 (active 상태)
  - 체크박스 클릭 → 하단 플로팅 버튼 오픈 (섹션 8 참조)
  - 소속 기업 셀 클릭 → `/companies` 이동 (행 클릭과 별개 동작)
- **정렬:**
  - 컬럼 헤더 클릭 시 오름차순/내림차순 토글
  - 기본 정렬: 이름 가나다순
- **페이지네이션:**
  - 하단에 페이지네이션 or 무한 스크롤
  - 한 페이지 기본 20개

### 5. 사이드 드로어 — 참여자 상세

#### 5-1. 드로어 기본 스펙

- **위치:** 우측에서 슬라이드 인
- **너비:** 480px ~ 560px (반응형 시 화면의 40%)
- **오버레이:** 반투명 배경 (클릭 시 드로어 닫힘)
- **애니메이션:** 우측에서 좌측으로 300ms ease-out

#### 5-2. 드로어 헤더

```
┌────────────────────────────────────────────────────┐
│  박진우                                       [>]  │
│  한빛테크 · 선임연구원                              │
└────────────────────────────────────────────────────┘
```

- **이름:** 20px bold, 좌측 정렬
  - 더블 클릭 시 인라인 편집 모드 진입
- **소속 기업 · 직위:** 14px, `text-gray-500`
  - 기업명 클릭 → `/companies` 해당 기업 드로어로 이동 (링크 스타일)
- **우측 아이콘:**
  - `[>]` 닫기 아이콘 → 드로어 닫기

#### 5-3. 참여자 정보 섹션

- **섹션 제목:** "참여자 정보"
  - 우측에 `[편집]` 연필 아이콘
- **표시 필드 (읽기 모드):**
  - 2컬럼 그리드 레이아웃

  ```
  소속 기업     한빛테크           직위       선임연구원
  연락처        010-1234-5678     이메일     jinwoo@hanbit.co.kr
  ───────────────────────────────────────────────────
  고용보험      ✅ 가입            업무 경력   5년차 ~ 10년차
  문서작성 역량  일부 작성 경험 있음
  ```

  - Label: 12px, 회색
  - Value: 14px, 검정, medium weight
  - 구분선 위: 기본 인적사항 / 아래: 네이버 폼 유입 데이터 (부가 정보)

- **편집 모드 진입 시:**
  - 각 Value가 input/select 필드로 전환
  - 소속 기업 → Combobox (기업 검색/선택, 이적 가능)
  - 직위 → text input
  - 고용보험 → 토글 스위치
  - 업무 경력 → select (`3년차 이하` / `3~5년차` / `5~10년차`)
  - 문서작성 역량 → select (5단계)
  - 하단에 `[저장]` `[취소]` 버튼 노출
  - `Enter` → 저장, `Esc` → 취소

#### 5-4. 교육 과정 이력 섹션

- **섹션 제목:** "교육 과정 이력"
  - 우측에 전체 수료 현황 뱃지: `수료 2/3`
- **과정 구분별 그룹핑:**
  - 과정 타입 헤더: 아이콘 + 텍스트 (기업 드로어와 동일 패턴)
    - `🟢 훈련비과정`
    - `🔵 지원비과정`
    - `🟡 공유개방 세미나`
  - 각 타입 아래 구분선 (divider, 점선)

- **세부 과정 행 (접힘 상태):**

  ```
  🟢 훈련비과정
  ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈
  ▸ 의료기기 품질관리 기초 및 실무     🟢 수료
  ▸ 스마트팩토리 실무                ⚪ 미수료

  🔵 지원비과정
  ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈
  ▸ 설계 지원 프로그램                 🟢 수료
  ```

  - 좌측: 펼침/접힘 화살표 (`▸` / `▾`)
  - 과정명: 14px, medium
  - 우측: 수료 상태 뱃지
  - Hover 시: 행 배경색 변경

- **세부 과정 행 (펼침 상태):**

  ```
  ▾ 의료기기 품질관리 기초 및 실무     🟢 수료
    ┌─────────────────────────────────────────────┐
    │  진행 기간    2026.05.12 ~ 05.14 (3일)      │
    │  교육 시간    24시간                          │
    │  수료 상태    🟢 수료        [상태 변경 ▼]   │
    │  수료일       2026.05.14                     │
    │  수료번호     CRT-2026-0042                  │
    │  신청일       2026.04.09                     │
    └─────────────────────────────────────────────┘
  ```

  - **표시 정보 (읽기 모드):**
    - 진행 기간: 시작일 ~ 종료일 (자동 계산 일수)
    - 교육 시간: N시간
    - 수료 상태: `🟢 수료` / `⚪ 미수료` + `[상태 변경 ▼]` 드롭다운
    - 수료일: date (수료 시 노출)
    - 수료번호: `certificateNo` (수료 시 노출)
    - 신청일: date (네이버 폼 제출 시점)
  - **`[상태 변경 ▼]` 동작:**
    - 미수료 → 수료 선택 시: 수료일 + 수료번호 입력 필드 자동 노출
    - 수료 → 미수료 선택 시: 확인 다이얼로그 ("수료 정보가 삭제됩니다")
    - 변경 즉시 API 호출 (인라인 저장, 별도 저장 버튼 없음)

### 6. 과정 연결 모달

- **트리거:** 드로어 내 교육 과정 이력 섹션 상단 `[+ 과정 연결]` 버튼
- **형태:** 센터 모달 (480px 너비)
- **목적:** 현재 참여자를 기존 세부 과정에 연결 (Enrollment 생성)

```
┌──────────────────────────────────────────────────┐
│  과정 연결 — 박진우                         [✕]   │
├──────────────────────────────────────────────────┤
│                                                  │
│  과정 구분                                        │
│  [훈련비과정 ▼]                                   │
│                                                  │
│  세부 과정                                        │
│  [GMP 심화 과정                            ▼]     │
│                                                  │
│  ※ 이미 등록된 과정은 목록에서 제외됩니다          │
│                                                  │
│  신청일 (선택)                                    │
│  [2026-04-23      📅]                             │
│                                                  │
├──────────────────────────────────────────────────┤
│                                  [취소]  [연결]   │
└──────────────────────────────────────────────────┘
```

- **과정 구분 드롭다운:** 훈련비 / 지원비 / 세미나
- **세부 과정 드롭다운:** 선택된 구분의 세부 과정 목록
  - 이미 등록된 과정은 비활성화 + `(등록됨)` 표시
- **신청일:** 선택 입력 (기본: 오늘 날짜)
- **`[연결]` 클릭:**
  - Enrollment 생성 (수료 상태 기본: 미수료)
  - 드로어 교육 과정 이력 섹션에 즉시 반영
  - 토스트: `"GMP 심화 과정에 연결되었습니다"`

### 7. 참여자 추가 모달

- **트리거:** 페이지 상단 `[+ 참여자 추가]` 버튼
- **형태:** 센터 모달 (480px 너비)
- **입력 필드:**

| 필드              | 필수 | 타입     | 비고                                                           |
| ----------------- | ---- | -------- | -------------------------------------------------------------- |
| 이름              | ✅   | text     |                                                                |
| 소속 기업         | ✅   | Combobox | 기업 검색/선택. 미등록 기업일 경우 `[+ 새 기업으로 등록]` 옵션 |
| 직위              | 선택 | text     |                                                                |
| 연락처            | ✅   | tel      | 전화번호 형식 검증                                             |
| 이메일            | ✅   | email    | 이메일 형식 검증                                               |
| 고용보험 가입여부 | 선택 | toggle   | 기본: 미선택                                                   |
| 업무 경력         | 선택 | select   | 3년차 이하 / 3~5년차 / 5~10년차                                |
| 문서작성 역량     | 선택 | select   | 5단계                                                          |

- **소속 기업 Combobox 동작:**
  - 타이핑 시 기업명 실시간 검색
  - 검색 결과에서 선택 → `companyId` 연결
  - `[+ 새 기업으로 등록]` 선택 시:
    - 기업 추가 미니 폼 인라인 노출 (기업명 + 소재지 + 대표명만, 최소 정보)
    - 등록 후 자동 선택됨
    - 협약 상태 기본: 미체결

- **하단:**
  - `[취소]` `[등록]`
  - 등록 완료 → 테이블에 즉시 반영 + 토스트 알림

#### 8-1. 플로팅 버튼 동작 정의

#### ✉ 이메일 발송 (Primary)

- **스타일:** `bg-green-600 text-white rounded-md px-4 py-2`
- **동작:** 클릭 시 이메일 발송 모달 오픈
- **모달 내용:**
  - 수신자 목록: 선택된 참여자의 **개인 이메일** 자동 채움
  - 제목 / 본문 입력 (템플릿 선택 가능)
  - 발송 대상 미리보기: `박진우(한빛테크), 김민지(한빛테크), 이태호(미래정밀)...`
- **이메일 미등록 참여자:** ⚠️ 경고 표시 ("이메일 미등록 참여자 N명 제외됨")

#### 📥 내보내기 (Secondary)

- **스타일:** `bg-gray-100 text-gray-700 rounded-md px-4 py-2`
- **동작:** 선택된 참여자만 필터링하여 엑셀 다운로드
- **내보내기 포함 정보:**
  - 참여자 기본 정보 (이름, 소속 기업, 직위, 연락처, 이메일)
  - 고용보험 / 경력 / 문서작성 역량
  - 등록 과정별 수료 상태, 수료일, 수료번호

#### ✕ 선택 해제 (Ghost)

- **스타일:** `bg-gray-100 text-gray-500 rounded-md px-4 py-2`
- **동작:** 모든 체크박스 해제 → 플로팅 바 slide-down 퇴장
- **키보드:** `Esc` 키로도 선택 해제 가능

#### 8-2. 체크박스 인터랙션 규칙

| 동작                      | 결과                                        |
| ------------------------- | ------------------------------------------- |
| 헤더 ☐ 클릭               | 현재 페이지 전체 행 선택/해제 토글          |
| 개별 행 ☐ 클릭            | 해당 행만 선택/해제                         |
| Shift + 클릭              | 마지막 선택 행 ~ 현재 클릭 행까지 범위 선택 |
| 행 본문 클릭 (이름 등)    | **체크 아님** — 사이드 드로어 열림          |
| 소속 기업 셀 클릭         | **체크 아님** — `/companies` 이동           |
| 체크 상태에서 드로어 열기 | 체크 유지, 드로어와 독립 동작               |
| 페이지 이동 시            | 선택 상태 초기화                            |
| 전체 선택 후 개별 해제    | 헤더 체크박스 → indeterminate(`─`) 상태     |

### 9. 상태 관리 및 데이터 흐름

- **참여자 테이블 데이터:**
  - 전역 상태 관리 (Zustand / Context)
  - 필터 탭 변경 → 서버 재조회 or 클라이언트 필터링
  - 보조 필터 (수료 상태, 고용보험) → 클라이언트 필터링
  - 검색 → 클라이언트 필터링 (데이터 적을 시) or 서버 검색
- **드로어 데이터:**
  - 행 클릭 시 `participantId`로 상세 조회 API 호출
  - 참여자 정보 + 교육 과정 이력 (Enrollments with SubCourse details) = 한 번에 fetch
  - 드로어 내 편집/상태 변경 시 → API 호출 → 성공 시 로컬 상태 업데이트 + 테이블 반영
- **교차 네비게이션:**
  - `/companies` → `/participants`: 기업 드로어 Popover의 `[상세보기 →]` 클릭 시
    - URL: `/participants?open=participantId`
    - 페이지 로드 후 해당 참여자 드로어 자동 오픈
  - `/participants` → `/companies`: 소속 기업 셀 또는 드로어 내 기업명 클릭 시
    - URL: `/companies?open=companyId`
    - 페이지 로드 후 해당 기업 드로어 자동 오픈
- **수료 상태 변경 시 연쇄 업데이트:**
  - 참여자 드로어에서 수료 상태 변경 → 참여자 테이블 수료 현황 컬럼 즉시 반영
  - 기업 드로어 교육 과정 섹션의 해당 참여자 수료 상태도 동기화 (다음 접근 시 최신 반영)

### 10. 반응형 및 예외 처리

- **반응형:**
  - 1280px 이상: 테이블 전체 컬럼 표시 + 드로어 480px
  - 1024px ~ 1279px: 이메일 컬럼 숨김 + 드로어 400px
  - 1024px 미만: 드로어가 전체 화면 오버레이
- **빈 상태:**
  - 참여자 0명 시: 일러스트 + "등록된 참여자가 없습니다" + `[+ 참여자 추가]` CTA
  - 검색 결과 0개 시: "검색 결과가 없습니다" 메시지
  - 드로어 내 교육 과정 이력 0개 시: "등록된 교육 과정이 없습니다" + `[+ 과정 연결]` CTA
- **에러 처리:**
  - API 실패 시: 토스트 에러 메시지
  - 네트워크 오류 시: 재시도 버튼
- **삭제 확인:**
  - 참여자 삭제 시 → 확인 다이얼로그
  - "삭제하면 모든 교육 과정 참여 기록도 함께 삭제됩니다" 경고
  - 과정 연결 해제 시 → "해당 과정의 수료 기록이 삭제됩니다" 경고
